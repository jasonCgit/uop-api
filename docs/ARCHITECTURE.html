Unified Observability Portal (UOP) Architecture Guide — High-level system design and integration overview

Detailed API specs: API-CURRENT — endpoint reference  |  API_DATA — schemas & payloads  |  OUTCOME-MEASURES — SRE metrics

1. Overview

Current State

Two repositories: uop-api (FastAPI backend) and uop-ui (React frontend). Backend currently uses mock data — all contracts are designed for live integration replacement.

Data Flow

  1. Frontend APPS: 81 applications with business metadata for filter cascading and tree navigation
  2. Backend APPS_REGISTRY: Same 81 apps with operational fields (incidents, region)
  3. Backend Enrichment: Computes health status bottom-up, attaches deployments, SLO metrics
  4. API Filtering: All endpoints accept filter query params and return scoped data
  5. Frontend Rendering: Pages fetch filtered data, render with scoped context, sync state to URL

6. External System Integrations

V12

Single source of truth for application catalog, technology hierarchy, deployments, and team roles (including SRE contacts).

  • Application metadata: name, SEAL, owner, SLA, risk ranking, classification, state
  • Technology hierarchy: LOB → CTO → CBT → Application
  • Application → Deployment mappings
  • Team roles tied to Application contacts

ERMA

Component dependency graph and cross-application relationship data, complementing V12.

  • Component dependency graph: nodes, edges, cross-application relationships
  • Business Process → Application mapping

Knowledge Graph

Platform, data center, and health indicator node data.

  • Platform nodes (GAP, GKP, ECS, EKS, AWS) with status
  • Data center nodes (NA, EMEA, APAC) with region and status
  • Health indicator nodes for RAG status computation

Unified Reporting API

Incident, problem, and change data correlated to applications.

  • Active incidents (P1/P2) with severity, status, timestamps
  • Incident history (30-day, 90-day trends)
  • MTTR, MTTA, resolution rate, escalation rate

Service Status & GEM/Cortex

SLO targets and actuals, error budgets, burn rates.

  • SLO target and current values per application/deployment
  • Error budget remaining and burn rate
  • Historical SLO performance for trend analysis

AURA AI Streaming API

AI-powered analysis, summarization, and chat with observability context.

  • AURA Summary (critical alerts, trend analysis, recommendations)
  • Blast Radius impact severity and executive summary
  • Interactive chat with SSE streaming

Detailed API endpoint specs are in API-CURRENT and API_DATA.

Filter System

ScopeBarUI controls → FilterContextReact context → Query StringURL encoding → API CallsFiltered endpoints

Filters use AND logic and cascade dynamically — selecting LOB = "AWM" restricts CTO dropdown to AWM CTOs only. Four filter groups: Business Hierarchy (LOB, Sub-LOB, Product Line, Product), Technology (CTO, CBT), Application (SEAL, Risk, etc.), Infrastructure (Deployment).

IssueImpact
